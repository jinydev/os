13강. 대용량 저장장치와 디스크 스케줄링

🎯 학습 목표

  1. 디스크 메커니즘의 한계 이해: 하드 디스크(HDD)의 탐색 시간과 회전 지연이 유발하는 물리적 병목 현상을 파악한다.
  2. 디스크 스케줄링 정복: 제한된 환경 속도에서 퍼포먼스를 뽑아내기 위한 최적의 헤드(Head) 이동 알고리즘(SSTF, SCAN, C-SCAN 등)을 이해한다.
  3. RAID 시스템 아키텍처: 데이터의 파손을 견뎌내고 병렬 전송 속도를 확보하기 위한 엔터프라이즈 서버 기반의 RAID 기술을 습득한다.
  4. 저수준 C언어 디렉터리 제어: 운영체제 메모리 상에서 디렉터리를 개방하고 파싱하는 로우레벨 제어 코드를 익힌다.



1. 물리적 탐색의 시간, 디스크 지오메트리 (Geometry)

하드디스크(HDD)는 메모리(RAM)와 달리 기계적으로 움직이는 암(Arm)과 회전하는 원판(Platter)으로 구성되어 있습니다. 메모리가 전자 펄스로 즉시 데이터를 뽑아내는 반면, 디스크는 헤드를 원하는 트랙까지 기계적으로 이동시켜야만 합니다.

디스크의 접근 시간(Access Time)은 다음의 세 가지 지연으로 구성됩니다:

  • 탐색 시간 (Seek Time): 헤드가 위아래로 움직여 타겟 실린더(트랙)를 찾는 여정. 가장 느리고 치명적인 병목 지점입니다.
  • 회전 지연 시간 (Rotational Latency): 디스크 원판이 뱅글뱅글 돌아 타겟 섹터가 헤드 밑으로 도달할 때까지 기다리는 시간.
  • 전송 시간 (Transfer Time): 실제 데이터를 읽어 메모리로 전송하는 시간.

운영체제의 디스크 스케줄링 알고리즘은 오직 저 끔찍한 첫 번째, “탐색 시간(Seek Time)을 어떻게 하면 최소화할까?” 에 모든 목적을 두고 설계되었습니다.



2. 디스크 스케줄링 (Disk Scheduling) : 헤드의 춤사위

대기열(Queue) 큐에 요청된 블록 번호가 “98, 183, 37, 122, 14, 124, 65, 67” 순으로 들어왔을 때, 이를 FCFS(먼저 온 순서)로 처리하면 헤드가 디스크 좌우를 미친 듯이 왕복 횡단하게 되어 시스템 효율이 바닥으로 추락합니다.

이를 해결하기 위해 운영체제는 궤적 자체를 통제합니다.

알고리즘별 특징

  • SSTF (Shortest Seek Time First):
    • 현재 헤드의 위치에서 가장 가까운 트랙부터 우선 처리합니다.
    • 단기적인 디스크 퍼포먼스는 가장 뛰어나지만, 멀리 있는 트랙은 영원히 서비스받을 수 없는 기아(Starvation) 상태에 빠질 수 있는 치명적 한계가 있습니다.
  • SCAN (엘리베이터 알고리즘):
    • 헤드가 디스크의 맨 끝(0)에서 반대편 끝(199)까지 마치 엘리베이터처럼 한 방향으로 이동하며 처리합니다. 벽을 치면 방향을 바꿉니다. 기아 현상이 완전히 해소됩니다.
  • C-SCAN (Circular SCAN):
    • SCAN과 비슷하지만 벽을 만나면 반대 방향으로 쓸고 오는 것이 아니라, 바로 출발점으로 텔레포트(Jump) 하여 다시 동일한 방향 한쪽으로만 스캔합니다. 균일한 대기 시간을 보장하여 가장 이상적입니다.

💡 (참고) SSD의 스케줄링: FTL

최신의 SSD(Solid State Drive)는 내부에 모터나 기계팔이 없습니다. 따라서 위에서 본 탐색 지연 파괴 알고리즘이 필요 없으며, FCFS 방식으로도 거의 균일한 성능을 냅니다. 하지만 SSD 플래시 블록은 “데이터를 덮어쓸 수 없고 반드시 블록을 통째로 깎아내야만(Erase) 다시 쓸 수 있다” 는 치명적인 전기적 특성이 있습니다. 이를 조율해 주는 것이 SSD 내장 컨트롤러인 FTL(Flash Translation Layer) 입니다.



3. RAID 아키텍처: 병렬화와 데이터의 무결성

RAID(Redundant Array of Independent Disks)는 여러 개의 저렴한 물리적 디스크를 묶어서, 운영체제(커널)에는 “1개의 거대한 논리 디스크” 처럼 속이면서 속도 향상과 데이터 백업을 동시에 달성하는 전설적인 시스템 규격입니다.

  • RAID 0 (Striping): 데이터를 여러 하드에 조각내어 동시 저장합니다. 속도가 기하급수적으로 빨라지지만, 하드 1개만 고장 나도 전체 데이터가 폐기물로 변합니다.
  • RAID 1 (Mirroring): 모든 쓰기 작업을 2개의 하드에 똑같이 기록합니다(복제). 1개가 물리적으로 불타버려도 즉시 가동 가능하므로 DB 서버에 필수적이나, 돈이 두 배로 듭니다.
  • RAID 5 (Parity): 최소 3개의 디스크를 요구하며 용량 낭비를 최적화한 실무 표준입니다. 복구 전용 암호화된 Parity 블록을 여러 드라이브에 분산 저장하여, 하나의 드라이브 손실을 감내하면서도 RAID 0과 같은 읽기 속도 이점을 공유합니다.



4. 디렉터리 파싱 (C언어 시스템 콜)

UNIX/Linux 아키텍처 설계에서 디렉터리 역시 궁극적으로는 그저 “하나의 파일”일 뿐입니다. 단, 평문 텍스트가 적혀있는 것이 아니라, [파일 이름 : i-node 번호]의 메타데이터 매핑 장부가 적혀 있는 특수한 파일입니다. 시스템 프로그래머는 다음과 같은 방식으로 이를 해독합니다.

#include <stdio.h>
#include <dirent.h>

int main() {
    DIR *dp;
    struct dirent *dirp;

    // 1. 디렉터리 포인터를 커널에서 획득하여 스트림을 개방한다
    if ((dp = opendir("/home/user/workspace")) == NULL) {
        return 1;
    }

    // 2. 디렉터리의 끝에 도달할 때까지 한 줄씩 읽어(readdir) 구조체로 반환받는다.
    while ((dirp = readdir(dp)) != NULL) {
        // 구조체 내부에 하위 파일의 이름과 i-node 정보가 들어있다.
        printf("i-node 번호: %ld | 이름: %s\n", dirp->d_ino, dirp->d_name);
    }

    closedir(dp);  // 메모리 누수 방지를 위한 자원 반환
    return 0;
}



📝 정리하기

  • 기계식 디스크 성능의 핵심 지표는 가장 오래 걸리는 탐색 시간(Seek Time)을 줄이는 헤드 제어 기술에 달려 있다.
  • 시스템의 안정적인 부하 분산을 위해서는 SSTF보다는 SCAN 유도체(C-SCAN, LOOK 등)가 훨씬 균일하게 작동한다.
  • 단일 디스크의 한계에서 벗어나는 RAID 컨트롤러 세팅을 통해 현대 서버는 IOPS 속도를 높이고 데이터 영속성을 쟁취한다.
  • 응용 프로그래머는 opendir(), readdir()을 조합한 반복 루프를 통해 커널 자료구조인 디렉터리 장부에 직접 접근할 수 있다.
