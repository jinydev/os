# 윈도우(Windows) 실무 개발 프레임워크

## 1. 교과목 개요

- **과목 소개**: 단순한 사무용/게임용 PC를 넘어, Windows OS가 어떻게 MS-DOS 시대부터 현대 클라우드 친화적인 하이브리드 엔진(WSL2)으로 진화했는지 그 근원부터 파고듭니다. 전통적인 DOS 명령어부터 객체지향의 끝판왕인 PowerShell, 닷넷(.NET) 생태계인 C# 프로그래밍 기초, 그리고 Windows 시스템 위에서 완벽한 프론트/백엔드 서버 런타임(PHP/Node.js/Python/Java 등)을 배포하는 전문 역량을 기릅니다.
- **학습 목표**: MS-DOS 시절의 레거시 명령어와 시스템 환경변수(PATH)를 완벽히 이해하고, 터미널 환경을 개선하며, 순수 Windows 네이티브에서 C# 앱은 물론 리눅스와 융합된(WSL2) 하이브리드 인프라를 통제할 수 있다.

3. Windows NT 등장: 기업용 서버 시장을 노린 진정한 커널(추상화 계층) 시스템

2. 백슬래시(`\`) 기호 경로 구분자의 역사적 기원 (리눅스와 차이점)

### 5주차
- **대주제**: 차세대 프레임워크, PowerShell 객체 셸
- **세부학습목표**: CMD 텍스트 스트리밍에 지친 관리자를 위한 마이크로소프트의 역작, 닷넷(.NET) 기반 객체 언어인 PowerShell을 배운다.

#### 📌 5-1. PowerShell 이란?
1. 텍스트 문자열(String)이 아닌 객체(Object) 파이프라인의 충격
2. `PS C:\>` 명령어의 기본 규칙: `동사-명사 (Verb-Noun)`
3. 스크립트 실행 권한(`ExecutionPolicy`) 보안 정책 해제하기

#### 📌 5-2. 스크립트 코딩과 객체 접근
1. `Get-Process | Where-Object WorkingSet -gt 1GB | Stop-Process` 메모리 학살 스크립트 작성
2. WMI와 CIM 클래스를 이용해 하드웨어 시스템 온도, CPU 코어 모델명 출력해내기
3. `Profile.ps1` 파일을 이용한 터미널 자동화 및 Oh-My-Posh 프롬프트 테마 꾸미기
