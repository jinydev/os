# 4. 서버 무정지 시스템의 심장, RAID 아키텍처

저렴하고 용량이 비교적 작은 여러 개의 디스크들 묶음을(Array) 리눅스 커널에서 **"완벽한 1개의 거대한 초고급 성능 디스크"**로 속여버립니다. 이를 통해 묶은 개수만큼 속도 스펙을 사기로 끌어올리고, 고장 시 서버 중단을 막는 생존 복구율을 구축하는 하드웨어 규격이 RAID 입니다.

![RAID 아키텍처](./img/os13_raid_architecture.svg)

## 핵심 RAID 단계

* **RAID 0 (Striping / 스트라이핑)**
  데이터 묶음 스트림을 여러 디스크에 동시 1/N 쪼개기 조각내어 병렬 송출합니다. IO 속도가 디스크 개수만큼 배수로 극대화되는 뽕맛이 있으나, 운용 중 디스크 중 단 한 개라도 뻑이 나면 해당 서버 묶음이 영구 완전 소각되어 날아갑니다. (백업용, 캐시 속도용으로만 씀)
* **RAID 1 (Mirroring / 미러링)**
  하드 2개에 무조건 쌍둥이(Clone) 거울 복제 기록을 강제합니다. 하버 하나가 불이 나 박살나도 남은 거울본으로 무중단 서비스가 구동되나, 구매 비용이 2배로 들며 용량 낭비가 뼈아픕니다.
* **RAID 5 (Parity 탑재 / 실무 표준)**
  데이터 트래픽 조각을 여러 하드에 분산하되, 각 하드 구석 자투리에 남은 디스크들을 결합한 XOR 연산을 꼬아 만든 비밀 암호 **패리티(Parity) 블록**을 은닉해 흩뿌려둡니다. 만약 운영 중 디스크 3개 묶음 중 1개가 돌연사로 박살나도, 즉결 패리티 방정식을 거꾸로 풀어서 소거된 데이터를 100% 확률로 그 즉시 수학적 재생 복원해 냅니다. 오늘날 실무의 코어 지표입니다.
