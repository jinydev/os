# 2. 데이터 전송 철학: Polling에서 Interrupt, DMA까지

CPU가 이 수많은 제어기(Controller)들과 통신하는 메커니즘은 3단계의 아득한 최적화 역사를 겪었습니다.

![I/O 데이터 전송 진화 과정](./img/os14_data_transfer_methods.svg)

## 1. 폴링 (Polling / Busy-Waiting)
CPU가 "도착했니? 완료됐니?" 하며 I/O 버퍼를 0.001초마다 반복 직접 감시합니다. 가장 원초적이고 무식한 방법이며 비싼 CPU 코어의 엄청난 연산 낭비입니다.

## 2. 인터럽트 (Interrupt)
CPU가 디스크에게 데이터 검색 작업 명령만 턱 던져두고 자신의 다른 할 일을 하러 파이프라인으로 돌아갑니다. I/O 작업이 끝나면 디스크 하드웨어 통제 칩상에서 전기 펄스 핀셋이 튀어 "임무 완료!"라는 **비동기 H/W 인터럽트**를 CPU 코어에 날려 운영체제가 즉결 처리(`ISR`)하게 합니다. 가장 흔한 운영체제의 핵심 근간이 자리를 잡습니다.

## 3. [최종 병기] DMA (Direct Memory Access) 아키텍처
10Gbps 광랜 인터넷이나 초당 기가바이트를 넘기는 Gen5 NVMe SSD가 쏘아붙이는 미친 속도의 전송 텍스트를 CPU가 인터럽트로 일일이 쫓아가서 RAM으로 퍼나르면 극심한 오버헤드 붕괴가 일어납니다. 

때문에 아예 전송 전용 모터 보좌관인 **DMA 제어기 칩**에게 버스 통제권을 통째로 위임해 버립니다! 
DMA 칩은 CPU 코어가 잠을 자건 딴짓을 하건 신경 안 쓰고, CPU 몰래 다이렉트로 메인 메모리(RAM)에 데이터를 한 번에 수십 MB 단위로 뭉터기 집어던져(Write) 놓습니다. 그리고 전송이 끝난 마지막에 딱 1번만 "대용량 전송 임무 완수" 인터럽트를 쏜 뒤 소멸하는 궁극의 **모바일/서버 I/O 최적화** 방식입니다.
