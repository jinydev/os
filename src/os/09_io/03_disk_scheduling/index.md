# 3. 스토리지의 숙명: 디스크를 향한 탐색 스케줄링

본질적으로 자성을 띤 기계 모터(Arm/Platter 구조)가 들어가는 전통적인 HDD 스토리지의 입출력 지연율(Latency)은 압도적으로 느립니다.

![디스크 지오메트리](./img/os13_disk_geometry.svg)

원하는 데이터 트랙으로 헤드를 움직이는 모터 이동 시간인 **탐색 시간(Seek Time)**이 결국 전체 처리 속도의 99%를 깎아 먹습니다. 이 탐색 이동 거리를 최소화하기 위한 하드웨어 경로 찾기 스케줄러가 커널에 존재합니다.

![디스크 스케줄링 알고리즘](./img/os13_disk_scheduling.svg)

* **FCFS**: 들어온 순서대로 모터를 움직입니다. 극도로 낭비가 심합니다.
* **SSTF (최단 탐색 시간 우선)**: 헤드 현 위치에서 가장 물리적으로 가까운 트랙 우선. 양옆만 빙빙 돌다가 저 멀리있는 트랙 섹터 요청은 영원히 버림받는 아사(기아 상태)가 필연적으로 발생.
* **SCAN (엘리베이터 스케줄링)**: 0 고도에서 출발해 199 끝단까지 벽을 칠 때까지 한 방향으로만 쓸어담고, 부딪히면 다시 튕겨 정반대로 돌아옵니다.
* **C-SCAN (이상적 결론)**: 엘리베이터가 벽을 치는 순간 다시 똑같이 돌아오지 않고, 시작점인 로비(0층 트랙) 초입으로 다이렉트 텔레포트 한 뒤(Cicular), 다시 **처음과 동일한 한쪽 방향으로만 쓸고 나갑니다.** 
  이렇게 하면 중간이든 양 끝단이든 전체 트랙에 대한 접근 대기 시간 평균이 가장 극도로 균일하게 변환되어 현대 운영체제의 기본 교리가 되었습니다.
