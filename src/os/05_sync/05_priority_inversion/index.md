# 5. [특수 심화] 우선순위 역전과 커널 상속 프로토콜

위와 같은 파괴적인 락킹 시스템이 운영체제의 스케줄러를 만나면 정말 기형적이고 추적하기 어려운 버그를 낳습니다.

RTOS(실시간 OS, 예: 화성탐사선 경로계산 시스템)에서 우선순위가 바닥인 하위 프로세스가 락을 잡고 있는 와중에, 가장 시급한 운전대 제어 핵심 프로세스가 이 동일한 자원을 요청하게 되면 락이 풀릴 때까지 영문도 모른 채 대기열에서 밀려나 무한정 블로킹되는 **우선순위 역전(Priority Inversion)** 사태가 터집니다.

![우선순위 역전 문제](./img/os06_54.svg)
![우선순위 상속 메커니즘](./img/os06_56.svg)

이를 해결하기 위해 현대 커널은 비정상 상태를 인지하고 즉시 강제 개입합니다. 대기하고 있는 상위 프로세스의 막강한 우선순위를, 현재 락을 빌려 사용 중인 하위 프로세스에게 덮어씌워 상속해주는 **우선순위 상속(Priority Inheritance)** 프로토콜을 터뜨립니다. 하급 프로세스가 이 상속받은 높은 우선순위를 바탕으로 다른 중간 프로세스들에게 방해받지 않고 초고속으로 작업을 끝내 락을 떨구게 만드는 궁극의 시스템 구명줄 아키텍처입니다.
