# 2. 하드웨어의 개입: 원자적 연산과 스핀락(Spinlock)

소프트웨어적인 진입 통제를 세팅하는 락(Lock) 변수 자체가 에러를 일으킬 수 있으므로, 현대 시스템은 하드웨어 실리콘 칩 차원에서 잘라질 수 없는 **원자적(Atomic) 연산 명령어** 조합을 제공합니다.

![TestAndSet Atomic Instruction](./img/os05_34.svg)

* **TestAndSet / Compare-and-Swap(CAS)**: 메모리의 예전 값을 읽어옴과 동시에 새로운 값으로 세팅하는 두 동작을 **절대 쪼갤 수 없는 하나의 Instruction**으로 처리합니다.
* **스핀락(Spinlock)의 한계**: 이를 이용해 락을 못 얻으면 `while(TestAndSet)` 무한 루프를 돌며 기다립니다(유저 모드). 이는 멀티코어에서 남이 곧장 풀어나가면 엄청난 속도를 내지만 길어질 경우 CPU 파워를 의미 없이 낭비합니다.
* **현대적 최적화 (Futex)**: 결국 성능에 민감한 C/C++ 게임 서버들은 OS 커널단으로 넘어가 대기(Block)하는 오버헤드 시간조차 아까워 처음엔 유저 모드의 이 CAS 루프로 짧게 뺑뺑이를 돌다 실패시에만 시스템 콜로 잠드는 퓨텍스(Futex) 메커니즘을 씁니다.
