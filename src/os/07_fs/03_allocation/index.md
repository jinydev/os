# 3. 물리적 디스크 블록 할당 기법 (Allocation)

데이터 조각(블록)들을 자성 디스크의 물리 트랙이나 SSD 셀 구역에 어떻게 심어야 성능이 가장 좋을까요? 세 가지의 발전 계보가 있습니다.

![블록 할당 방식 비교](./img/os12_file_allocation.svg)

## 1. 연속 할당 (Contiguous Allocation)
카세트테이프처럼 무조건 비어있는 연속된 통짜 공간을 찾아 1열 횡대로 데이터를 죽 기록합니다. 
물리적으로 이어져 있으므로 한 방향으로 긁어버리는 순차 읽기(Sequential Read) 속도는 광속의 한계를 뚫어버리지만, 여러 파일이 지워지고 새로 쓰인 이후 생기는 수많은 수백 메가의 '구멍(빈칸/외부 단편화)' 때문에 정작 나중에 생긴 10GB짜리 큰 파일을 담지 못하고 디스크 전체 수명이 쪼그라듭니다.

## 2. 연결 할당 (Linked / 커스텀 FAT)
작은 블록 조각 안에 마치 링크드 리스트처럼 "다음 위치 데이터가 쓰인 조각 블록 번호 꼬리표"를 달아 전 세계(?) 전체 트랙에 흩뿌립니다. 
아무리 큰 파일도 조각내어 빈칸에 다 심어버리니 외부 단편화 낭비는 완벽히 없어집니다. 치명타는, 영화의 중간 50분 구간을 임의로 짚어 읽고(Random Access) 싶을 때, 무조건 1분부터 시작하는 기차 꼬리표를 순차적으로 타고 내내 이동해야 하므로 하드 분산 탐색 시간(Seek Time)이 영구적으로 붕괴됩니다. MS-DOS의 FAT 시스템이 이를 개량한 물건이었습니다.

## 3. 색인 할당 (Indexed Allocation)
앞선 연결 할당의 탐색 붕괴를 타파하기 위해 별도의 인덱스 책갈피 전용 블록을 한 개 희생시켰습니다. 
데이터가 제아무리 디스크 사방팔방으로 쪼개져 흩어져있어도, 해당 파일 전담 색인표 한 번만 뒤지면 즉각 타겟 주소에 다이렉트 엑세스가 가능합니다. (이 사상은 현대 UNIX의 i-node 아키텍처 근간 철학으로 채택되었습니다.)
