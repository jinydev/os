# 5. [전공 심화] 저널링(ext4)과 CoW 기반 차세대 파일시스템(APFS/ZFS)

여러분이 수십 페이지의 워드(Word) 문서를 한창 작성하여 저장(Save) 커맨드를 누르는 그 찰나, 갑자기 정전이 되어 전원 코드가 뽑혀버리면 수백 개의 페이지 노드 트리들이 수정되다 만 채로 저장된 파일 시스템은 즉결 파괴될까요? 이 참사를 무효화하는 궁극의 백업/커밋(Commit) 트랜잭션기가 운영체제 역사상 최고봉의 예술상입니다.

## ext4 / NTFS 엔진: 저널링 (Journaling)
기존의 파일시스템은 디스크를 긁으며 덮어쓰다 전기 코드가 뽑히면 트리 연결이 박살나 다신 부팅 안 되는 파일 무결성 오류를 뿜었습니다 (Windows 95/98 시절).

현대의 ext4는 다릅니다. 램 데이터를 고속 디스크 트리에 반영하여 덮기 전에, 디스크의 외곽에 숨겨둔 별도의 `Journal` 외상 장부 구역(Log 구역)에 **"나 지금부터 시스템 권한으로 i-node 581번 A 트리를 편집할 거야. 완료될 때까지 기다려."**라고 이 일기예보(로그 증거) 트랜잭션을 무조건 선제 기록합니다. 
도중에 전기가 폭파되어 나갔어도 다음 재부팅 시 저널 장부의 무결성 검증을 굴려서 누락된 잔여 트랜잭션을 소화(Recovery)해 파일을 손상 없이 방어하는 **방탄 커밋 방어벽**입니다.

## 차세대 비파괴 복제엔진: Copy-on-Write (ZFS, Mac APFS)
저널링을 넘어, 최신 OS(특히 애플의 Mac이나 Linux 도커 컨테이너)는 아예 '덮어쓰기'라는 파괴 개변 자체를 죄악으로 규정짓고 소멸시켰습니다!

기존 데이터 블록 원본을 단 1바이트도 부수거나 건드리지 않고 고결하게 놔둔 채, 널찍한 빈공간의 아예 새 블록에 수정 변경 내역을 따로 써내려 갑니다. 그러곤 최종 병합이 완료될 때 **루트 포인터 맵핑 링크 줄 하나만 옛 블록에서 새 블록 쪽으로 방향성을 싹 교체**해버립니다.

* **스냅샷의 파괴력**: 덕분에 이 최첨단 파일 시스템 환경에서는 단 1초 만에 전체 디스크 시점 백업을 투리 증명하듯 찍어버리는 스토리지 가상화 **스냅샷(Snapshot)** 권능을 지닙니다. 시스템 전체에 랜섬웨어가 걸려 부서져도 과거 수백 개의 포인터 맵핑 이력이 저장되어 있으므로 **어제 오후 5시 시점으로 클릭 한 번에 즉각 데이터 회귀 마법**을 발현하는 갓(God) 계층에 오르게 되었습니다.
