# 2. 가상 메모리의 마법: 페이징과 세그먼테이션

지긋지긋한 외부 단편화를 근본적으로 없애기 위해 현대 OS는 프로세스를 믹서기로 잘게 썰어버리는 아이디어에 도달했습니다.

## 페이징 (Paging) 시스템
물리 메모리를 일정한 **4KB** 조각인 **프레임(Frame)**으로 네모반듯하게 썰고, 프로세스 가상 메모리도 완전히 동일한 4KB 크기의 **페이지(Page)**로 썹니다.

![페이징 아키텍처](./img/os10_paging.svg)

이를 통해 램의 빈 공간이 제아무리 이빨 빠진 듯 찢어져 있어도 조각들을 차례로 끼워 넣을 수 있어 **외부 단편화가 시스템에서 100% 소멸**합니다! 커널은 `페이지 테이블`이라는 장부를 거쳐 맵핑을 추적합니다.

## 진화된 현대 혼합: 페이지화된 세그먼테이션

하지만 데이터는 데이터끼리, 코드는 코드끼리 논리적/의미적 단위로 자를 수 있는 **세그먼테이션(Segmentation)**은 타 프로세스로부터의 강력한 영역 보안 통제(Read-Only 등)와 라이브러리 공유 능력을 지니는 장점이 있습니다.

현대의 리눅스와 윈도우는 이 두 마리 토끼(단편화 방지와 보안)를 다 잡기 위해, 프로세스를 논리적(세그먼트 보안 영역)으로 나누고 다시 그 세그먼트 블록들을 물리적인 페이지(4KB) 크기로 썰어 적재하는 **페이지화된 세그먼테이션(Paged Segmentation)**을 실전 메모리 코어로 채택했습니다.
