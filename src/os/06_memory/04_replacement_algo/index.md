# 4. 페이지 대치(Replacement)와 퇴출 알고리즘

만약 요구 페이징을 하다가, 시스템의 물리적 RAM에 빈 4KB 프레임이 단 한 칸도 안 남고 100% 꽉 차버리면 어떻게 될까요? 
운영체제는 기존에 쾌적하게 자리 잡고 있던 페이지 중 하나를 찍어서 잠시 하드디스크 구석기(Swap 영역)로 쫓아내고(**희생자 선출: Victim**) 자리를 강제로 확보해야 합니다. 그 희생양을 고르는 결정 로직이 대치 알고리즘입니다.

![알고리즘 비교표](./img/os11_lru_algorithm.svg)

## FIFO의 한계와 벨라디의 모순 (Belady's Anomaly)
무조건 큐에 일찍 들어온 녀석을 순서대로 쫓아내는 가장 멍청한 방식입니다. 
경우에 따라 돈을 들여 물리적인 램(RAM) 메모리 프레임을 3칸에서 4칸으로 증설해 줬음에도 불구하고, 타이밍이 괴상하게 꼬여 오히려 에러율(Page Fault)이 전보다 더 치솟아버리는 말도 안 되는 아노말리(모순/역설)가 수학적으로 관측되기에 현대 OS 디자인에서는 절대 쓰지 않습니다.

## LRU (Least Recently Used) 아키텍처
적중(Hit)될 때마다 해당 메모리 블록을 최상위 계급 공간으로 올려놓고, 반면 쫓아낼 희생자를 고를 땐 하단의 **'가장 오랫동안 단 한 번도 사용되지 않은 녀석'**을 무자비하게 썰어버리는 극단적 실용성 최적화 알고리즘입니다. 
미래의 참조 패턴을 알 수 없는 컴퓨터 공학 세계에서 과거의 로그를 바탕으로 가장 완벽에 가까운 실전 지표를 내주는 현대 OS용 코어 알고리즘 설계사입니다.
