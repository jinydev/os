# 3. [전공 심화] 주소 공간 독립성과 fork()의 철학

운영체제는 도대체 어떻게 프로세스라는 거대한 컨테이너 격리 감옥을 계속해서 새로 찍어낼까요? 리눅스는 거대한 메모리 공간과 권한을 맨땅에서 새로 창조하는 대신, 매우 극단적이고 안전한 유닉스 철학을 구사합니다. 빈 공간이 아니라 항상 기존의 '부모' 프로세스를 복제하여 생성하는 기형적 증식 구조를 허가합니다.

![fork, exec, exit, wait 논리](./img/os03_32.svg)

시스템 콜의 아름다운 협상 과정은 다음과 같습니다:

1. **`fork()` 콜**: 부모 프로세스 자신과 완벽히 동일한 메모리 레이아웃을 통째로 복제한 자식 프로세스를 분기합니다. (현대 커널은 복제 페널티를 막기 위해 변경 사항만 복사하는 **CoW: Copy-on-Write** 엔진을 활용합니다)
2. **`exec()` 콜**: 이 자식 프로세스의 심장 공간 안에서 내 몸의 기억(코드)을 싹 덮어씌워 (Overwrite) 전혀 다른 새로운 프로그램(`/bin/ls` 이나 카카오톡 등)으로 껍데기만 남긴채 변이합니다!
3. **`wait()` & `exit()` 콜**: 자식 프로세스는 자신의 임무를 달성한 뒤 종료하며 시체가 되고, 부모 프로세스는 이를 감시하다 수거해 고이 좀비(Zombie) 상태인 자식 PID를 완전히 시스템에서 소멸(Reap)시켜줍니다.

만약 C 코드 내에 `exec()` 함수 지시자가 터진 순간, 바로 아래줄에 기입해 놓은 `printf("hello")` 구문은 무의미합니다. 부모의 몸에서 자식으로 잉태했지만, `exec`를 맞은 순간 그 프로그램 코드의 영혼 자체가 타겟 지시물로 치환(Replacement)되었기 때문에 절대 도달할 수 없습니다.
