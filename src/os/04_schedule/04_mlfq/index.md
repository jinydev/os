# 4. 실전 상용 알고리즘: 다단계 피드백 큐 (MLFQ) 구조

SJF는 구현이 불가능하지만, 현대 OS는 **과거의 행동 기록을 토대로 미래를 예측**해 흉내 내는 전략을 채택했습니다.

## 우선순위 스케줄링의 부작용: 기아 현상
가벼운 프로세스에 우선순위를 몰아주면 무거운 배치 작업들은 평생 CPU를 잡아보지 못하고 굶어 죽어버리는 **아사 현상 / 기아 상태(Starvation)**에 돌입합니다. 
OS는 가벼운 형벌 구제로 시간이 지난 놈들에게 강제로 계급장을 올려 우대해 주는 **에이징(Aging)** 특수 논리를 더합니다.

## 다단계 피드백 큐 (Multi-Level Feedback Queue) 구조
단일 큐인 라운드-로빈(RR)을 넘어, 현재 윈도우와 유닉스(Solaris) 등은 여러 계단의 큐를 복합/계층적으로 설계했습니다.

1. **상위 큐 응답 보장**: 사용자 클릭 대기 큐(예: 브라우저/키보드 셸 입력)는 **최상위 등급 큐**로 지정되어 짧은 CPU 시간만을 빼앗아 즉각적인 UI 응답을 보장합니다.
2. **강등(Demote) 페널티**: 반면 CPU를 100% 끊임없이 혹사시키는 무거운 게임/인코딩 프로세스 등은 1단계에서 주어진 Time Quantum을 다 썼을 때 OS가 괘씸죄를 물어 **하위 큐로 점차 강등** 시켜 우선순위를 연속적으로 박탈합니다.
3. **패자 부활전 (Aging)**: 방치된 맨 밑바닥 큐 작업도 Aging 타이머를 통해 주기적으로 상위 큐로 한 번씩 끌어올려 주어 전체의 밸런스를 잡습니다.
