# 2. 선점형 타임 퀀텀 제어와 성능 평가 측정 기준

## 선점성(Preemptive)과 비선점(Non-Preemptive)
과거 MS-DOS 같은 환경은 특정 코드가 무한루프 `while(1)`에 빠지면 시스템 전체가 마비되었습니다(`비선점`). 그러나 현대 OS 단기 스케줄러는 1초를 수백 조각으로 쪼개서(Time Quantum, 타임 슬라이스) A-B-A로 교대 할당합니다. 

이때 하드웨어 타이머 칩셋이 **인터럽트를 터뜨려 강제로 CPU 제어권을 환수**하는 `선점형(Preemptive)` 방식을 쓰기에 어떠한 민폐 프로그램이라도 컴퓨터를 개별적으로 동결시킬 수 없습니다.

## 스케줄러 모델 평가 지표 (Metrics)
스케줄링 알고리즘이 우수한지를 평가하는 수학적 기준입니다.
![성능 지표 측정선](./img/os07_42.svg)

* **대기 시간 (Waiting Time)**: 큐에서 아무것도 하지 못하고 CPU를 기다린 순수 딜레이 타임 합계.
* **응답 시간 (Response Time)**: 마우스 클릭 후 화면에 최초 반응이 오기까지 걸리는 UX 최중요 지표. (현대 데스크톱 스케줄러의 타겟)
* **반환 시간 (Turnaround Time)**: 큐에 도착 후 계산이 모두 끝나 시스템에서 소멸할 때까지의 생애 총합.
